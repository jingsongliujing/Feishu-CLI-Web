# lark-reliable-scenes

这个 skill 用来稳定处理高成功率飞书 CLI 场景。它不是新的 CLI 域，而是一组稳定编排规则：先查真实 ID，再写入；能用 shortcut 就用 shortcut；多步骤任务必须完成所有动作后才能结束。

## 通用稳定规则

1. 任何写操作都必须等待用户确认，确认后再执行。
2. 不要编造 `ou_`、`oc_`、`docx`、`base`、`file_token`、`event_id` 等 ID。
3. 用户名先用 `lark-cli contact +search-user --query <name>` 搜索，拿到 `open_id` 后再继续。
4. 群名先用 `lark-cli im +chat-search --query <group>` 搜索，拿到 `chat_id` 后再继续。
5. 私信用 `lark-cli im +messages-send --user-id <open_id> --text <text> --as user`。
6. 群消息用 `lark-cli im +messages-send --chat-id <chat_id> --text <text> --as user`。
7. 创建群先搜索所有成员 open_id，再 `lark-cli im +chat-create`。
8. 创建日程优先用 `lark-cli calendar +create --summary --start --end --attendee-ids`。
9. 找共同空闲时间优先用 `lark-cli calendar +suggestion`，群可以直接使用 `oc_` chat_id 作为 attendee id。
10. 用户要求“会议号/会议链接/会议详情”时，创建日程必须带 `vchat.vc_type=vc`，并从返回结果提取 `meeting_url` 或会议号再通知。
11. Excel/CSV 导入多维表格必须用 `lark-cli drive +import --type bitable --file <path> --name <title> --as user`。如果返回 `next_command`，继续执行 `lark-cli drive +task_result ...` 直到拿到最终结果。
12. 多步骤任务不能在第一个写操作成功后提前结束。例如“导入多维表格并发送到群”必须完成导入、拿链接、搜索群、发送群消息。
13. 群会议通知优先发纯文本，包含时间、会议号、会议链接；不要手写 interactive 卡片 JSON。
14. 本地文件路径保持用户原样输入，例如 `./docs/data_all_20260329_201138.xlsx`。

## 场景分解

### 创建日程

1. 搜索所有参与人。
2. 计算开始/结束时间。
3. `calendar +create` 创建日程。

### 创建群

1. 搜索所有成员。
2. 用成员 open_id 创建群。
3. 返回群名和 chat_id。

### 发私信

1. 搜索联系人。
2. 用 open_id 发文本消息。

### 创建文档并写内容

1. 如果用户给了明确标题，用标题创建文档。
2. 如果内容是“今天的十条 AI 新闻”这类生成内容，先让模型生成简洁 markdown，再 `docs +create --markdown`。

### 创建多维表格

1. 创建 base。
2. 如需写入组织人员，先查联系人或用户列表，再按字段写入记录。
3. 如需导入 Excel，优先走 `drive +import --type bitable`。

### 群里找共同会议时间

1. 搜索群拿 `oc_`。
2. `calendar +suggestion` 查询共同空闲。
3. 创建带飞书视频会议的日程。
4. 把会议号、会议链接、时间发回群。

### 多人找共同会议时间并分别通知

1. 搜索所有人员 open_id。
2. `calendar +suggestion` 查询共同空闲。
3. 创建带飞书视频会议的日程。
4. 分别给每个人发送会议信息。

## 失败时修复

- 找不到人：换姓名全称重试一次，仍失败就停止并列出未找到人员。
- 找不到群：换去掉括号或引号的群名重试一次。
- 发 interactive 卡片失败：降级为纯文本消息。
- 导入 Excel 返回异步 ticket：执行返回的 `next_command`。
- 创建会议但没拿到会议号：从 `meeting_url` 解析；解析不到就发送会议链接和 event_id。
